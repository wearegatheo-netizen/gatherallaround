# GatherO 전체 기능 QA 보고서 — 2026-07-07

대상: `index.html`(단일 파일 SPA, ~13,000줄) + `functions/` + `supabase/functions/` + `bgm-extension/` + PWA.
방법: 6개 영역 정적 분석 + 오프라인 브라우저 스모크 테스트(인라인 스크립트 문법 검증, 화면 전환 검증). 실행 환경상 외부 CDN/Supabase가 차단되어 실 DB 연동 동작은 미검증.
라인 번호는 커밋 `0834cb7` 기준. 수정 전 `grep -n` 재확인 권장.

## 1. 인증 / 가입 (Auth)

**Major**
- Kakao 로그인/가입 경로가 `state.user`에 `authType:'kakao'`를 안 넣음(2163 등). 이메일 경로는 `authType:'email'` 설정. 현재는 로그아웃 분기가 우연히 맞지만 취약. → 명시 설정.
- `handleLogout`(2188)이 `supabaseClient` null 체크 없이 `auth.signOut()` 호출.
- **밴드 가입 시 profile insert 실패를 조용히 삼킴**(10410) — 실패해도 성공 토스트. auth 유저만 생기고 profile 없는 고아 상태. → `bandSignupResult`로 오류 표시 + 성공 토스트 중단.
- 밴드 가입 중복 ID 사전 체크 없음(10392) — Supabase 원문 영어 에러 노출.

**Minor**
- 규약 위반: `handleResetPw`(11142), `submitKakaoSignup/Link`, `checkEmailUserApproval`, `handleLogout` 등에서 `alert()` 사용(인라인 result div 규약 대상 폼).
- `handleFindId`(11120) anon key로 이름+전화 조회 → 마스킹 이메일 반환. RLS가 anon select 허용해야 동작 = PII 노출 정책 서버 검증 필요.
- 이메일 가입 profile insert 실패 시 auth 유저 고아화(11203) — 서버 트리거로 정리 필요(클라 해결 불가).
- `showPendingScreen`(11160)이 main-content/band-main-content/community 등 미숨김 → 비정상 경로서 이전 화면 노출 가능. → `_hideAllScreens()` 먼저.

**정상 확인**: XSS 없음(이름은 textContent/escape), 중복제출 가드 전 폼에 존재(`_signupBusy` 등), 폼 전환 시 result div 초기화 대체로 양호.
